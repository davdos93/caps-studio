# GitHub Pages einrichten

## Bereitgestellte URLs

Nach erfolgreicher Aktivierung:

- Startseite: `https://BENUTZERNAME.github.io/caps-studio/`
- Beta: `https://BENUTZERNAME.github.io/caps-studio/beta/`
- Stable: `https://BENUTZERNAME.github.io/caps-studio/stable/`

## Veröffentlichungsquelle

- Branch: `main`
- Ordner: `/ (root)`

## Wichtiger Hinweis

Bei GitHub Free ist Pages nur aus öffentlichen Repositorys verfügbar.
Für Pages aus einem privaten Repository ist GitHub Pro, Team oder Enterprise erforderlich.

## Sicherheitsprinzip

CAPS enthält keine API-Schlüssel und keine serverseitigen Geheimnisse.
Projektinhalte werden lokal im jeweiligen Browser gespeichert.
Die Website-Dateien können öffentlich erreichbar sein, die lokal erstellten Projekte
werden dadurch nicht automatisch auf GitHub hochgeladen.
